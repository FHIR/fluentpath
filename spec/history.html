March 31, 2018 1.1.0 Normative Ballot. Changes:
  • Added terminology operators to FHIR appendix
  • Increased precedence of is/as operators
  • Added .aggregate() operator
  • Clarified support for namespaces in FHIRPath and its usage contexts (FHIR, CQL, V2)
  • Relaxed conversion failures to return null instead of throw an error
  • Added support for quantity comparisons
  • Relaxed definition of code equivalence (only uses code and system, not version or text/display)
  • Added $index accessor to iteratio contexts
  • Added profile validation operators to FHIR appendix

See changes from ballot to release for the spec or the grammar
STU1 sequence
Dec 6, 2016 1.0.0 STU1 Release. Changes:
  • Rename from FluentPath to FHIRPath for trademark reasons
  • Added string concatenate (&) operator
  • Added URI and Media Types
  • Added combine operator to support combining lists without removing duplicates
  • Added anyFalse and allFalse operators
  • Clarified lexical elements for whitespace, comments, case-sensitivity, etc.
  • Added ability to reference case-insensitive models, and note about models specifying alternative collection offsets
  • Added a usage appendix for HL7 V2
  • Added an implementation appendix
  • Added examples and clarifying documentation throughout

See changes from ballot to release for the spec or the grammar

  • refine notions around 3 value Boolean logic
  • clarify use & syntax of constants
  • add date operations
  • many minor changes to functions
  • provide test cases for FHIR implementations
  • added support for reflection
  • improved support for extensions and profiles in FHIR
